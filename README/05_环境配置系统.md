# 05 · 环境配置系统（MDP 设计）

> **目标**：理解 IsaacLab 的 Manager-Based 设计，学会修改奖励函数、观测、动作、终止条件。

---

## 1. MDP（马尔可夫决策过程）与 IsaacLab 的对应

强化学习的 MDP = (S, A, R, T, γ)，在 IsaacLab 中：

```
MDP 元素          IsaacLab 组件               配置类
──────────────────────────────────────────────────────
S  状态/观测   ←  ObservationManager      ←  ObservationsCfg
A  动作        ←  ActionManager           ←  ActionsCfg
R  奖励        ←  RewardManager           ←  RewardsCfg
T  终止条件    ←  TerminationManager      ←  TerminationsCfg
-  随机扰动    ←  EventManager            ←  EventCfg
-  场景物体    ←  Scene (ArticulationCfg) ←  CartpoleisaaclabSceneCfg
```

## 2. 整体配置类层级

```
CartpoleisaaclabEnvCfg  (你的任务配置)
├── scene: CartpoleisaaclabSceneCfg
│   ├── ground: AssetBaseCfg          ← 地面
│   ├── robot: ArticulationCfg        ← CartPole 机器人（CARTPOLE_CFG）
│   └── dome_light: AssetBaseCfg      ← 环境光照
│
├── observations: ObservationsCfg
│   └── policy: PolicyCfg(ObsGroup)
│       ├── joint_pos_rel: ObsTerm    ← 关节位置（4096×2）
│       └── joint_vel_rel: ObsTerm    ← 关节速度（4096×2）
│       → 拼接后输出 [4096, 4]
│
├── actions: ActionsCfg
│   └── joint_effort: JointEffortActionCfg
│       ← 把 NN 输出 × 100 施加到 slider_to_cart 关节
│
├── events: EventCfg
│   ├── reset_cart_position: EventTerm  ← 重置时随机小车位置
│   └── reset_pole_position: EventTerm  ← 重置时随机杆子角度
│
├── rewards: RewardsCfg
│   ├── alive: RewTerm                  ← 存活 +1.0
│   ├── terminating: RewTerm            ← 终止 -2.0
│   ├── pole_pos: RewTerm               ← 杆子偏移 -1.0
│   ├── cart_vel: RewTerm               ← 小车速度 -0.01
│   └── pole_vel: RewTerm               ← 杆子速度 -0.005
│
└── terminations: TerminationsCfg
    ├── time_out: DoneTerm              ← 超时（truncated）
    └── cart_out_of_bounds: DoneTerm    ← 小车越界（terminated）
```

## 3. 每个 Manager 的工作原理

### 3.1 ObservationManager

```
step() 时自动调用:

ObservationManager.compute()
    ↓
遍历 PolicyCfg 中每个 ObsTerm
    ↓
joint_pos_rel: 调用 mdp.joint_pos_rel(env)
    → 返回所有关节位置 [4096, 2]
    (slider_to_cart: 小车位置, cart_to_pole: 杆子角度)
    
joint_vel_rel: 调用 mdp.joint_vel_rel(env)
    → 返回所有关节速度 [4096, 2]
    
concatenate_terms=True:
    → torch.cat([pos, vel], dim=-1) → [4096, 4]
    
noise (enable_corruption=False, 跳过)
    ↓
返回 {"policy": tensor[4096, 4]}
```

**如果只想观测特定关节：**
```python
# cartpoleisaaclab_env_cfg.py
joint_pos_rel = ObsTerm(
    func=mdp.joint_pos_rel,
    params={
        "asset_cfg": SceneEntityCfg("robot", joint_names=["cart_to_pole"]),
        # 只返回杆子角度，不返回小车位置
    },
)
# 输出变为 [4096, 1] 而不是 [4096, 2]
```

---

### 3.2 RewardManager

```
step() 时自动调用:

RewardManager.compute()
    ↓
遍历 RewardsCfg 中每个 RewTerm
    ↓
alive:      mdp.is_alive(env) × 1.0
            → [4096, 1] 全1（环境未结束）
            
terminating: mdp.is_terminated(env) × -2.0
            → [4096, 1] 已终止的环境=-1，其余=0
            
pole_pos:   mdp.joint_pos_target_l2(env, target=0.0, ...) × -1.0
            → wrap_to_pi(pole_angle)² 
            → [4096, 1] 偏离越大惩罚越多
            
cart_vel:   mdp.joint_vel_l1(env, ...) × -0.01
            → |cart_velocity|
            → [4096, 1] 速度越大惩罚越多
            
pole_vel:   mdp.joint_vel_l1(env, ...) × -0.005
            → |pole_velocity|
            
求和: rewards[4096, 1] = Σ(term_value × weight)
    
还要乘以 rewards_shaper_scale=0.1 (在 record_transition 里)
```

def joint_pos_target_l2(env: ManagerBasedRLEnv, target: float, asset_cfg: SceneEntityCfg):

# 配置使用
pole_pos = RewTerm(
    func=mdp.joint_pos_target_l2,
    weight=-1.0,

### 3.4 EventManager（重置随机化）

控制每次 `env.reset()` 时如何随机初始化：

```python
reset_cart_position = EventTerm(
    func=mdp.reset_joints_by_offset,
    mode="reset",          # 只在 reset() 时触发
    params={
        "asset_cfg": SceneEntityCfg("robot", joint_names=["slider_to_cart"]),
        "position_range": (-1.0, 1.0),   # 小车初始位置 ∈ [-1, 1] 米
        "velocity_range": (-0.5, 0.5),   # 初始速度 ∈ [-0.5, 0.5] m/s
    },
)
```

**mode 选项：**
- `"reset"` → 环境重置时触发
- `"startup"` → 首次启动时触发
- `"interval"` → 按 interval 步数触发

## 4. CartPole 机器人结构

```
CartPole USD 模型关节结构:

World
└── Robot (ArticulationCfg)
    ├── slider_to_cart  (棱柱关节 PrismaticJoint)
    │   ← 小车沿 X 轴滑动
    │   ← 动作施加在这里（力矩/力）
    │   ← ObsTerm: joint_pos_rel + joint_vel_rel
    │
    └── cart_to_pole   (旋转关节 RevoluteJoint)
        ← 杆子绕轴旋转
        ← 奖励计算: pole_pos (角度² 惩罚)
        ← 终止检测: 无（靠 alive + terminating 奖励引导）
        ← ObsTerm: joint_pos_rel + joint_vel_rel
```

**关节命名非常重要！** 配置中的 `joint_names` 必须与 USD 文件中的关节名匹配：

```python
# 关节名不匹配时的报错：
# Scene entity with key 'robot' not found. 
# Available Entities: ['terrain', 'robo', 'ground', 'dome_light']
```

## 6. 仿真时序配置

```
sim.dt = 1/120 秒  ← 物理更新频率（推荐值）
decimation = 2     ← 每个控制步包含几个物理步

时序图：

物理时间线:  ─────┬──────┬──────┬──────┬──────┬──────┬──────
           t=0   t=1/120 t=2/120 t=3/120 t=4/120 ...
                  ↑PhysX   ↑PhysX  ↑PhysX  ↑PhysX
                  
控制时间线:  ───────────────┬──────────────────┬────────────
                           step=0              step=1
                          (施加动作,           (施加动作,
                           覆盖两个物理步)      覆盖两个物理步)
                           
step_dt = 2/120 = 1/60 秒 ≈ 16.67ms (60Hz 控制频率)

render_interval = decimation = 2
→ 每个控制步渲染一次（不是每个物理步，节省GPU）
```

## 7. 如何修改环境：实战指南

### 修改奖励函数

```python
# cartpoleisaaclab_env_cfg.py → RewardsCfg

# 添加新的奖励项（用官方函数）
cart_pos = RewTerm(
    func=mdp.joint_pos_target_l2,
    weight=-0.5,        # 小车位置也要保持在中心
    params={
        "asset_cfg": SceneEntityCfg("robot", joint_names=["slider_to_cart"]),
        "target": 0.0,
    }
)

# 修改权重（更重视保持杆子直立）
pole_pos = RewTerm(
    func=mdp.joint_pos_target_l2,
    weight=-5.0,        # 从 -1.0 改为 -5.0
    params={"asset_cfg": SceneEntityCfg("robot", joint_names=["cart_to_pole"]), "target": 0.0},
)

# 删除奖励项：直接注释掉对应的 RewTerm 即可
```

### 修改观测空间

```python
# 添加杆子的绝对角度（不归一化）
class PolicyCfg(ObsGroup):
    joint_pos_rel = ObsTerm(func=mdp.joint_pos_rel)    # 保留原有
    joint_vel_rel = ObsTerm(func=mdp.joint_vel_rel)
    
    # 新增：只观测杆子位置
    pole_angle_only = ObsTerm(
        func=mdp.joint_pos_rel,
        params={"asset_cfg": SceneEntityCfg("robot", joint_names=["cart_to_pole"])},
    )
    # 注意：输出从4维变为5维，skrl 会自动感知 observation_space
```

### 修改终止条件

```python
# 添加杆子角度终止（杆子倒下超过45度就结束）
from isaaclab.managers import TerminationTermCfg as DoneTerm
import math

pole_fallen = DoneTerm(
    func=mdp.joint_pos_out_of_manual_limit,
    params={
        "asset_cfg": SceneEntityCfg("robot", joint_names=["cart_to_pole"]),
        "bounds": (-math.pi/4, math.pi/4),    # ±45度
    },
)
```

### 自定义奖励函数

```python
# mdp/rewards.py
import torch
from isaaclab.envs import ManagerBasedRLEnv
from isaaclab.managers import SceneEntityCfg
from isaaclab.assets import Articulation

def pole_upright_bonus(env: ManagerBasedRLEnv, asset_cfg: SceneEntityCfg) -> torch.Tensor:
    """杆子越直立奖励越高（余弦函数）"""
    asset: Articulation = env.scene[asset_cfg.name]
    pole_angle = asset.data.joint_pos[:, asset_cfg.joint_ids]
    # cos(0) = 1（直立），cos(±π) = -1（倒下）
    return torch.cos(pole_angle).squeeze(-1)

# 在 cartpoleisaaclab_env_cfg.py 中注册
upright_bonus = RewTerm(
    func=mdp.pole_upright_bonus,
    weight=2.0,
    params={"asset_cfg": SceneEntityCfg("robot", joint_names=["cart_to_pole"])},
)
```

## 8. 配置验证：常见错误和解决方法

| 错误 | 原因 | 解决 |
|------|------|------|
| `Scene entity with key 'robot' not found` | `asset_name="robot"` 但场景里定义的是 `robo` | 检查 SceneCfg 中的变量名 |
| `Joint 'xxx' not found` | joint_names 写错了 | 打开 USD 文件查看关节名 |
| `observation_space mismatch` | 修改了观测后忘记更新网络输入维度 | skrl 会自动推断，重启训练即可 |
| `rewards tensor shape mismatch` | 自定义奖励函数返回了错误shape | 确保返回 `[num_envs]` 或 `[num_envs, 1]` |
