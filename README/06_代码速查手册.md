## 2. 常用命令速查

```bash
# ──── 训练 ────
# 基础训练
python scripts/skrl/train.py --task Template-Cartpoleisaaclab-v0

# 指定并行环境数
python scripts/skrl/train.py --task Template-Cartpoleisaaclab-v0 --num_envs 64

python scripts/skrl/play.py --task Template-Cartpoleisaaclab-v0 --num_envs 4

# 录制视频
python scripts/skrl/play.py --task Template-Cartpoleisaaclab-v0 --video --video_length 300

# ──── 工具 ────
# 列出所有注册的任务
python scripts/list_envs.py

# Zero-action 测试（验证环境配置）
python scripts/zero_agent.py --task Template-Cartpoleisaaclab-v0

# Random-action 测试
python scripts/random_agent.py --task Template-Cartpoleisaaclab-v0
```

## 3. YAML 配置关键字段速查

```yaml
# skrl_ppo_cfg.yaml 完整注释版

seed: 42                    # 随机种子

models:
  separate: False           # False=共享网络，True=独立网络
  policy:
    class: GaussianMixin    # 策略：高斯（连续动作）
    clip_actions: False     # 是否截断动作到动作空间范围
    clip_log_std: True      # 截断 log_std
    min_log_std: -20.0      # log_std 下限
    max_log_std: 2.0        # log_std 上限（exp(2)≈7.4）
    initial_log_std: 0.0    # 初始标准差=1.0（exp(0)=1）
    network:
      - name: net
        input: OBSERVATIONS # 关键词：[OBSERVATIONS, ACTIONS, STATES_ACTIONS]
        layers: [32, 32]    # 隐藏层维度
        activations: elu    # 激活函数：[elu, relu, tanh, sigmoid]
    output: ACTIONS         # 关键词：[ACTIONS, ONE]
  value:
    class: DeterministicMixin  # 价值：确定性（单值输出）
    ...
    output: ONE             # ONE = 输出1维

memory:
  class: RandomMemory
  memory_size: -1           # -1 = 自动设置为 rollouts 值

agent:
  class: PPO
  rollouts: 32              # ★ 关键：多少步更新一次
  learning_epochs: 8        # ★ 每批数据重复学习几遍
  mini_batches: 8           # ★ 分几个小批次
  discount_factor: 0.99     # γ 折扣因子
  lambda: 0.95              # λ GAE系数
  learning_rate: 5.0e-04    # 初始学习率
  learning_rate_scheduler: KLAdaptiveLR  # 自适应调整
  learning_rate_scheduler_kwargs:
    kl_threshold: 0.008     # KL散度目标值
  state_preprocessor: RunningStandardScaler    # 观测归一化
  value_preprocessor: RunningStandardScaler    # 价值归一化
  grad_norm_clip: 1.0       # 梯度裁剪
  ratio_clip: 0.2           # ε PPO截断系数
  value_clip: 0.2           # 价值截断系数
  clip_predicted_values: True  # 是否截断价值预测
  entropy_loss_scale: 0.0   # 熵奖励（0=不用）
  value_loss_scale: 2.0     # 价值损失缩放
  rewards_shaper_scale: 0.1 # ★ 奖励整体缩放（防数值爆炸）
  time_limit_bootstrap: False  # 超时时是否bootstrap
  experiment:
    directory: "cartpole_direct"   # 日志目录名

trainer:
  class: SequentialTrainer
  timesteps: 4800           # ★ 总训练步数（4800/32=150次更新）
  environment_info: log     # 记录 info 中的哪个key
```

## 4. IsaacLab 常用 MDP 函数

### 观测函数（来自 isaaclab.envs.mdp）

```python
# 关节位置（相对于默认位置）
mdp.joint_pos_rel(env, asset_cfg=SceneEntityCfg(...))
# 返回: [num_envs, num_joints]

# 关节速度（相对）
mdp.joint_vel_rel(env, asset_cfg=SceneEntityCfg(...))
# 返回: [num_envs, num_joints]

# 关节位置（绝对值）
mdp.joint_pos_limit_normalized(env, asset_cfg=SceneEntityCfg(...))
# 返回: [num_envs, num_joints]，归一化到[-1,1]
```

### 奖励函数

```python
# 存活奖励（未终止=1，终止=0）
mdp.is_alive(env)
# 返回: [num_envs]

# 终止惩罚（终止=-1，未终止=0）
mdp.is_terminated(env)
# 返回: [num_envs]

# 关节位置目标L2损失（距目标位置的平方和）
mdp.joint_pos_target_l2(env, target: float, asset_cfg: SceneEntityCfg)
# 返回: [num_envs]

# 关节速度L1范数（速度绝对值之和）
mdp.joint_vel_l1(env, asset_cfg: SceneEntityCfg)
# 返回: [num_envs]

# 关节速度L2范数（速度平方和）
mdp.joint_vel_l2(env, asset_cfg: SceneEntityCfg)
# 返回: [num_envs]
```

### 终止函数

```python
# 超时终止
mdp.time_out(env)
# 返回: [num_envs], bool

# 关节位置越界
mdp.joint_pos_out_of_manual_limit(
    env, 
    asset_cfg: SceneEntityCfg,
    bounds: tuple[float, float]    # (min, max)
)
# 返回: [num_envs], bool
```

### 事件函数

```python
# 关节位置偏移重置（以默认值为中心随机偏移）
mdp.reset_joints_by_offset(
    env,
    asset_cfg: SceneEntityCfg,
    position_range: tuple[float, float],
    velocity_range: tuple[float, float],
)

# 关节位置直接重置到指定范围
mdp.reset_joints_by_scale(...)
```

## 5. configclass 关键用法

```python
from isaaclab.utils import configclass

@configclass                   # ① 装饰器，类似 @dataclass
class MyEnvCfg(ManagerBasedRLEnvCfg):
    
    # ② 字段定义（类型注解 + 默认值）
    scene: MySceneCfg = MySceneCfg(num_envs=4096, env_spacing=4.0)
    
    # ③ 嵌套配置，直接访问
    # env_cfg.scene.num_envs
    # env_cfg.sim.dt
    
    def __post_init__(self) -> None:   # ④ 在所有字段初始化后执行
        self.decimation = 2
        self.sim.dt = 1/120
        self.sim.render_interval = self.decimation  # ← 依赖其他字段，放这里
```

## 6. skrl 关键 API

```python
# ──── Runner ────
runner = Runner(env, agent_cfg)    # 一键初始化
runner.run()                       # 开始训练
runner.run(mode="eval")            # 推理模式
runner.agent                       # 获取 agent 实例
runner.trainer                     # 获取 trainer 实例

# ──── Agent ────
agent.act(states, timestep, timesteps)
# 返回: (actions, log_prob, outputs_dict)

agent.load("path/to/checkpoint.pt")    # 加载权重
agent.set_running_mode("eval")         # 推理模式（关闭 dropout 等）
agent.set_running_mode("train")        # 训练模式

# ──── Memory ────
memory.add_samples(states=..., actions=..., rewards=..., ...)
memory.get_tensor_by_name("advantages")    # 读取
memory.set_tensor_by_name("advantages", tensor)  # 写入
memory.sample_all(names=["states","actions",...], mini_batches=8)
# 返回: list of mini-batch tuples

# ──── GaussianMixin ────
# act() 返回: (actions, log_prob, {"mean_actions": mu})
# 推理时用 mean_actions（确定性），训练时用 actions（随机）
outputs = agent.act(obs, 0, 0)
actions = outputs[-1].get("mean_actions", outputs[0])  # play.py 的做法
```

## 7. 调试技巧

```python
# 打印 Memory 中的数据 shape
print(memory.get_tensor_by_name("states").shape)      # [32, 4096, 4]
print(memory.get_tensor_by_name("advantages").shape)  # [32, 4096, 1]

# 检查共享网络
print(self.policy is self.value)                           # False（不同实例）
print(self.policy.net_container is self.value.net_container)  # True（共享参数！）

# 检查 reward 是否合理（在 record_transition 加断点）
print(f"rewards mean: {rewards.mean():.4f}, std: {rewards.std():.4f}")

# 检查 advantage 是否归一化
adv = memory.get_tensor_by_name("advantages")
print(f"adv mean: {adv.mean():.4f}, std: {adv.std():.4f}")
# 应该接近 mean≈0, std≈1

# 观察学习率变化（KLAdaptiveLR）
print(f"lr: {runner.trainer.agents.scheduler.get_last_lr()}")
```

## 8. TensorBoard 监控

```bash
# 启动 TensorBoard
tensorboard --logdir logs/skrl

# 然后浏览器打开 http://localhost:6006
```

**关键指标说明：**

| 指标 | 正常范围 | 含义 |
|------|---------|------|
| `Loss / Policy loss` | 趋近 0 | 策略损失（越小越好） |
| `Loss / Value loss` | 趋近 0 | 价值损失（越小越好） |
| `Policy / Standard deviation` | 逐渐降低 | 策略越来越确定 |
| `Learning / Learning rate` | 浮动 | KLAdaptiveLR 自适应 |
| `Reward / Total reward (mean)` | 趋于增大 | 平均总奖励 |
| `Episode / Length` | 趋于增大 | 平均 episode 长度 |

## 9. 常见超参数调整指南

### 任务收敛慢
```yaml
rollouts: 16           # 从32改小，更频繁更新
learning_rate: 1.0e-3  # 增大学习率
timesteps: 10000       # 增加训练步数
```

### 训练不稳定
```yaml
rollouts: 64           # 增大，更多数据
learning_rate: 1.0e-4  # 减小学习率
grad_norm_clip: 0.5    # 更激进的梯度裁剪
```

### 想要更好的探索
```yaml
entropy_loss_scale: 0.01   # 添加熵奖励（鼓励随机性）
initial_log_std: 0.5        # 初始标准差更大（exp(0.5)≈1.65）
```

### 网络太小（欠拟合）
```yaml
# 在 skrl_ppo_cfg.yaml 修改网络结构
network:
  - name: net
    input: OBSERVATIONS
    layers: [64, 64, 64]   # 从[32,32]改为[64,64,64]
    activations: elu
```
